POLÍTICA DE VERSIONAMENTO

Modelo: Semantic Versioning (MAJOR.MINOR.PATCH)

Regras:
- MAJOR: mudanças grandes (arquitetura ou refatoração)
- MINOR: novas funcionalidades (novos módulos como Paciente, Frota)
- PATCH: correções de bugs

Aplicação no projeto:
v1.0.0 - estrutura inicial backend + frontend
v1.1.0 - CRUD de Pacientes e integração Angular
v1.2.0 - CRUD de Medicamentos e Ocorrências
v1.3.0 - CRUD de Frota + melhorias UI
v1.3.1 - correções gerais

Controle:
- Versionamento via tags GitHub
- CHANGELOG atualizado a cada release
